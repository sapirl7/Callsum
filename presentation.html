Как работает

Три шага от аудио до готового отчёта

STEP 01
Транскрипция
Распознавание речи с высокой точностью. Поддержка русского языка, длинных записей до 2 часов.
Whisper large-v3 — OpenAI, запущен локально
STEP 02
Диаризация
Определение кто говорит в каждый момент времени. Автоматическая разметка спикеров по голосу.
Pyannote 3.1 — speaker diarization
STEP 03
AI-анализ
Генерация структурированного саммари: обсуждённые темы, задачи с ответственными и дедлайнами, приоритеты.
Llama 3.1 8B — Meta, запущена на вашем GPU

Ключевое преимущество

Ваши данные никуда не утекают

Все ML-модели работают на выделенном GPU. Аудио обрабатывается внутри изолированного контейнера и не передаётся третьим лицам.

✅
Whisper — локально

Модель транскрипции запускается внутри Docker-контейнера на вашем GPU

✅
Pyannote — локально

Диаризация происходит на том же сервере, модель встроена в образ

✅
Llama 3 — своя LLM

Open-source модель от Meta. Никаких API-вызовов к внешним сервисам

✅
S3 — шифрование AES-256

Все файлы в облаке зашифрованы. Доступ только через presigned URLs

❌
Нет вызовов OpenAI API

Аудио не отправляется в ChatGPT, Claude или другие облачные LLM

❌
Нет внешних транскрипций

Речь не расшифровывается на серверах Google, Amazon или Microsoft

Архитектура

Serverless + GPU on demand

Два облака, чёткое разделение: AWS для оркестрации, RunPod для GPU-вычислений.

☁️

AWS — Оркестрация

Lambda — Telegram bot, обработка webhook
API Gateway — HTTPS endpoint
S3 — хранение аудио (AES-256, versioning)
DynamoDB — метаданные задач, rate limiting
Secrets Manager — токены и ключи
CloudWatch — логи, метрики, алармы

⚡

RunPod — ML вычисления

Serverless GPU — RTX 3090 / A4000
Docker — изолированный контейнер
Whisper — транскрипция (large-v3)
Pyannote — диаризация (3.1)
Llama 3 — AI-анализ (70B)
Auto-scaling — масштабирование по нагрузке

dataflow User → Telegram → AWS Lambda → S3 upload ↓ RunPod GPU (Whisper → Pyannote → Llama) ↓ S3 result → Lambda callback → Telegram → User

Развертывание

Четыре этапа до production

Этап 1

AWS инфраструктура

$ cd infrastructure/terraform $ cp terraform.tfvars.example terraform.tfvars $ terraform init && terraform apply → Сохранить api_gateway_url
Этап 2

ML-сервис (RunPod)

$ cd runpod_service $ docker build --build-arg HF_TOKEN=... \ -t user/callsum-ml:latest . $ docker push user/callsum-ml:latest → Создать Serverless Endpoint в RunPod
Этап 3

Связать компоненты

# В terraform.tfvars добавить: runpod_api_key = "ваш_ключ" runpod_endpoint = "https://api.runpod.ai/..." $ terraform apply -auto-approve
Этап 4

Webhook Telegram

$ curl -X POST \ "https://api.telegram.org/botTOKEN/setWebhook" \ -d '{"url": "API_GW_URL", "secret_token": "..."}' → Бот готов к работе!
Требования Terraform ≥ 1.0 · AWS CLI · Docker · Python 3.10+ · RunPod аккаунт ($10) · HuggingFace Token · Telegram Bot Token
